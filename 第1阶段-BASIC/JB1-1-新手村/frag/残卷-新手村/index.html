非Maven项目打Jar包

武技：使用IDEA将非Maven管理的项目打成jar包

  • File - Project Structure - Project Settings - Artifacts
  • 单击加号 - JAR - From Modules with dependicies
  • 在打开的对话框中指定jar包输出的位置，无需添加文件名。
  • 点击应用OK完成配置。
  • 回到主界面 - Build - Build Artifacts
  • 选择你的jar包，点击 build，完成jar包导出。
  • 如果需要写README.txt等附属文件，需要在classpath下创建。

GitHub

  • GitHub是用于版本控制和协作的代码托管平台，是一个开源的分布式的版本控制系统，只支持git这一种唯一版本库。
  • GitHub从08年4月上线，一直到18年6月被微软收购。
  • 登录 GitHub官网 注册GitHub账号并登录GitHub。
  • 创建GitHub仓库：一个仓库对应一个项目。

    • 勾选 Initialize this repository with a README，为你的项目生成一个README文件。

      • 151.101.109.194 github.global.ssl.fastly.net

从GitHub上clone项目到本地

心法： 你可以直接点击Download下载，但仍建议使用Git命令方式进行clone操作。

武技：

  • 复制远程库地址：https://github.com/joe-zhou-hub/joe-start.git
  • 在本地新建一个空白文件夹。
  • 在文件夹中右键开启Git命令行。
  • git clone <远程地址>
    • 该命令可以完整地将远程项目拷贝到本地。
    • 该命令可以自动完成了 git init 的工作。

从GitHub上pull项目到本地

  • git pull origin master 从远程获取最新版本并合并到本地，相当于依次执行如下两个命令：
    • git fetch 取回更新。
    • git merge 将更新内容合并到本地。
  • 如果远程库存在代码且和本地库无关联，会出现报错：
    • 报错内容：refusing to merge unrelated histories
    • 解决方案：git pull origin master --allow-unrelated-histories，该命令允许无关联合并。
    • 弹出的合并日志编辑器可以编写或直接 :wq 跳过。

使用Git命令行绑定远程库

武技：

  • 在Github上创建一个远程库 joe-start：
    • 远程库地址：https://github.com/joe-zhou-hub/joe-start.git
  • 在代码区编写文件start.txt，并且将文件提交到本地仓库分支上。
    • git add start.txt
    • git commit -m "首次提交" start.txt
  • 绑定远程仓库
    • git remote add origin 远程路径
    • origin 是远程仓库别名，随便起，建议默认。
  • 查看远程的仓库绑定信息
    • git remote -v
  • push操作之前需要先pull，保证数据一致性：
    • git pull origin master --allow-unrelated-histories
  • 将文件push到远程库 origin 的 master 分支上。
    • git push -u origin master
  • 填入用户名和密码。

赵四push本地项目

武技： 赵四将本地项目push到GitHub中

  • VCS - Import Into Version Control - Create Git Responsirty

解决代码冲突

概念：

  • 解决冲突的方法：编辑文件删除冲突代码，并重新提交。
  • 避免冲突的方法：
    • 多人修改同一文件会导致冲突，所以尽量分工明确。
    • 使用IDEA可以自动检测更新。
    • 提交前先更新（push前先pull）。

SVN

搭建SVN服务端

  • 安装SVN服务端：在欢迎界面直接下一步：
    • 勾选许可，下一步。
    • 选择 VisualSVN Server and Adminstration Tools 选项。
    • 勾选 Add Subversion command-line tools to the PATH environment variable，下一步。
    • 选择 Standard Edition 版本。
    • 填写 Location：SVN家目录。
    • 填写 Repositories：SVN仓库目录，下一步。
    • 点击 Install 安装。
    • 勾选 Start VisualSVN Server Manager 以启动SVN服务端。
    • res: VisualSVN-Server-3.5.0-x64.msi
  • 启动SVN服务端：安装结束之后默认是开启状态：
    • 点击 VisualSVN Server(Local)，然后看到菜单条上的开启按钮和暂停按钮。
    • 如果直接关闭面板，服务器仍是处于启动中状态，只有点击暂停按钮才可以停止服务器运行。
  • 创建用户：
    • 右键 Users -> Create Users：填写账号和密码。
  • 创建仓库：
    • 右键 Repositories -> Create New Repositories：选择 Regular FSFS respository... 选项。
    • 填写 respository name：仓库名。
    • 选择 Empty repository(recommended option) 空仓库。
    • 选择 All Subversion users have Read/Write access 所有客户端开放读写权限，点击create创建。
  • 访问仓库：
    • 右键仓库，选择 Copy URL to clipboard 以复制仓库地址。
    • 浏览器直接访问仓库地址，注意将计算机名称换成IP地址。

搭建SVN客户端

  • 安装SVN客户端，俗称小乌龟：
    • command line client tools 必须勾选，否则在idea上集成svn的时候会找不到 svn.exe 而报错。
    • 先点击右下角 Browse 浏览路径，再勾选 command line client tools，否则无法浏览路径。
    • 客户端默认的路径为C:\Program Files，这个路径因为存在空格，所以是不生效的。
    • res: TortoiseSVN-1.9.7.27907-x64-svn-1.9.7.msi

客户端同步仓库

  • 在硬盘上新建一个空文件夹test
  • 右键test文件夹，选择SVNCheckout，目的是将这个文件夹和服务器端的仓库文件夹同步。
  • 在test文件夹中，右键选择update，此时你会发现仓库中有的你这个文件夹中也会有，如此则成功完成同步。

更新资源

  • 新建：在你的test文件夹中新建资源，然后选择update，再commit则可以完成一个资源的提交。
  • 更新：在你的test文件夹中改动仓库中的资源，然后选择update，再commit则可以完成一个资源的修改。

武技：将工作空间中的项目共享到SVN中

  • 点击 VCS -> Import into Version Control 选择 Share Project(Subversion)...，将项目分享出去。
  • 点击加号连接SVN服务器，填写服务器URL
    • 下方的 Define share target 选择第二项，会默认为你生成一个文件夹。
    • 在 Commit Comment Prefix 中填写操作注释。
  • 注意要计算机名换成127.0.0.1。
  • 默认选择1.8 format即可，点击 OK 即可。
  • 在认证界面填写账号密码。
  • 勾选 Save credentials 保存认证。
  • 点击 OK 完成认证。
  • 分享完成，点击 OK 后，你会发现你的项目文件都变成了绿色（绿色代表未提交）
  • 是否将vcs.xml文件也分享到SVN上，这里选择 Cancel 取消。
  • 至此，你的项目已经分享到你的SVN服务器上了：
    • 若服务器上看不见这个项目，刷新即可。
    • 但是分享上去的只是一个空的文件夹，这是因为项目在上面，但是项目中的内容还未曾提交。
    • 我们可以通过右击项目 Subversion -> Commit Directory 来完成项目的提交操作。
    • 除第一次操作外，提交之前必须要先更新 Update Directory...，提交成功后的文件，将不再为绿色。
