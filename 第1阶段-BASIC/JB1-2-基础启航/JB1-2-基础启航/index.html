心法: 文档注释的优点

文档注释的优点描述
易修改文档注释和代码处于同一个Java文件中，修改代码的同时，顺便就修改了文档

EP03. 计算机计算原理

心法: 计算机计算原理

  • 计算机底层计算时，先将操作数都转为补码，然后再计算，最后将计算结果转回原码并输出其真值。
  • 原码: 一个数字的二进制形式，也叫机器数，原码最高位为符号位，0正1负。
  • 真值: 原码对应的十进制值。
  • 反码: 正数的反码同原码，负数的反码是其原码按位取反的结果，符号位不变。
  • 补码: 正数的补码同原码，负数的补码是其反码加1的结果，符号位不参与进位:
    • 补码的补码就是原码。
    • 可以使用 Integer.toBinaryString() 来查看一个数的补码。

武技: 测试4 + 6底层计算原理

  1. 开发测试方法 ConstantTest -> test4Plus6():
public class ConstantTest {

    /** 4 + 6 过程分析 */
    @Test
    public void test4Plus6() {
        System.out.println("4的补码: " + Integer.toBinaryString(4));
        System.out.println("6的补码: " + Integer.toBinaryString(6));
        /*
         * 模拟: 4 + 6 过程分析
         * 原码: 00000000 .. 00000100: 4原码
         * 反码: 00000000 .. 00000100: 正数同原码
         * 补码: 00000000 .. 00000100: 正数同原码
         * 原码: 00000000 .. 00000110: 6原码
         * 反码: 00000000 .. 00000110: 正数同原码
         * 补码: 00000000 .. 00000110: 正数同原码
         * 计算: 00000000 .. 00001010: 4的补码和6的补码相加的结果
         * 反码: 00000000 .. 00001010: 正数同原码
         * 补码: 00000000 .. 00001010: 正数同原码
         * 真值: 10
         */
        System.out.println(4 + 6);
    }
}

CH04. 变量与代码沼泽

心法: 变量

  • 在整个运行过程中可以发生变化的量叫变量。

EP01. 变量的声明方式

心法: 变量的声明方式

  • 变量声明就是在JVM内存中申请一块指定类型，指定名称的区域（盒子）的过程:
    • 声明格式: 变量类型 变量名

  • 相同类型的变量可以在同一行声明，用逗号分隔。
  • 仅声明未赋值的变量不能输出和计算。

EP03. 变量的命名规范

心法: 变量的命名规范

  • 变量名只能由数字/字母/下划线和$符号组成，数字不能开头。
  • 变量名不能使用关键字: Java系统自己用的，官方的，有着特殊功能和意义的单词，如 public/static 等。
  • 变量名不能使用保留字: Java系统没有使用到，但是也不让我们使用的单词，如 const/goto 等。
  • Java中的3个直接量也不能使用，包括 true/false/null。
  • 方法名: 小驼峰，一般使用动词，如 getName()/setName()/getAge()/setGender()。
  • 变量名: 小驼峰，一般使用名词，如 name/age/gender。
  • 只要像一个正常人一样去起名，就不会错！

EP04. 代码的整洁之道

心法: 代码沼泽困境

  • 糟糕的代码带来的困境就是代码沼泽，花时间保持代码整洁不但有关效率，还有关生存。

心法: 代码的整洁之道

  • 命名的时候多费心，且一旦发现有更好的名字，就换掉旧的。
  • 言到意到，见名知意: 如果你起的名字还得需要用到注释来补充说明的话，那它就不怎么合格。
  • 别在乎变量名的长度: 变量名越长描述的信息越多，但有趣的是只要短名称足够清楚，就比长的好。
  • 避免误导性的名字，如用userList来指定一组用户，无论该变量是不是List类型，改成users都会更好。
  • 别在变量名中使用 l，I 和 O，容易和数字 1 和 0 混淆。
  • 单词 variable 永远不该出现在你变量名中，单词 table 永远不应该出现在表名中。
  • 尽量别为变量名添加前后缀，因为快速阅读代码时，大脑根本就看不见前缀后缀。
  • 使用能读出口的变量名，避免 yyyy_MM_dd_hh_mm_ss 这种变量名。

CH05. 基本数据的类型

心法: 基本数据类型

  • Java中的基本数据类型一共有8个。
  • JDK11版本支持使用 var 自动推断局部变量基本/引用类型，建议在Lambda表达式中使用。

EP01. 基本类型之整数

心法: 基本数据类型 - 整数型

  • 整数型包括 byte(1)/short(2)/int(4)/long(8) 四种。
  • 整数常量默认为int型，为整数常量附加 l/L 后缀可使其默认为long型。
  • 整数常量赋值前需要接受JavaC的范围检查，通过可直接赋值，即满足范围的整数可直接赋值给byte/short型变量。
  • 若整数常量本身超出了其默认类型的存值范围，则JavaC直接报错。
  • long型值可以使用 _ 作为千分位分隔符，不影响值本身。

武技: 测试基本数据类型 - 整数型

  1. 开发测试方法 BasicTypeTest -> testInteger():
package basic;

/** @author lsx */
@SuppressWarnings("all")
public class BasicTypeTest {

    /** 4个整数类型 */
    @Test
    public void testInteger() {

        // 字节型: 占1字节，存值范围 -2^7 ~ 2^7-1 (-128 ~ 127)
        byte numA = 100;
        System.out.println(numA);

        // 短整型: 占2字节，存值范围 -2^15 ~ 2^15-1
        short numB = 10000;
        System.out.println(numB);

        // 整数型: 占4字节，默认值，存值范围 -2^31 ~ 2^31-1 (-2147483648 ~ 2147483647)
        int numC = 10000000;
        System.out.println(numC);

        // 长整型: 占8字节，存值范围 -2^63 ~ 2^63-1
        // 为整数常量附加 `l/L` 后缀可使其默认为long型
        long numD = 200000000L;
        System.out.println(numD);

        // 100000000000: 整数默认为int型常量，但该值已经超出int型的最大存储范围，报错
        // 100000000000L: 通知javac我是一个long型常量，该值未超出long型的最大存储范围，通过
        System.out.println(100000000000L);

        // long型值可以使用 `_` 作为千分位分隔符，不影响值本身
        System.out.println(100_000_000_000L);
    }
}

EP02. 基本类型之浮点

心法: 基本数据类型 - 浮点型

  • 因为Java支持科学计数法，所有小数点的位置是可以随意浮动的，故而成为浮点数:
    • 如 3.5e1 等于 0.35e2 等于 0.035e3 等。
  • 浮点型包括 float(4)/double(8) 两种:
    • 浮点常量默认为double型，为浮点常量附加 f/F 后缀可使其默认为float型。
  • 浮点数常量天生不精确，所以赋值前无法接受JavaC的范围检查:
    • 即所有的浮点数都无法直接被float型变量接收。

        System.out.println(new BigDecimal("2.00").subtract(new BigDecimal("1.10")));

EP05. 基本类型的转换

  • 八大基本数据类型之间，除了boolean之外，都是可以互相转换的:
    • 口诀: 由小到大自动转，由大到小需强转。
    • 格式: B b = (B)A
  • 浮点数转成整数时，会对浮点数向下取整。
  • 整数转成浮点数时，会在整数后面补充 .0 作为小数部分。
  • char类型可以使用其Unicode字符编号来参与转换。

         * 模拟: 2147483647 + 1 的过程
         * 原码: 01111111 11111111 11111111 11111111

EP03. 正则表达式专题

心法: 正则表达式 Regular Expression (RE)

  • 正则表达式是一个字符串表达式，用于对字符串进行匹配，对空格敏感。
  • RE正则字符:
    • 普通字符: 基础字符，包含数字和字母等，必须被 [] 包裹。
    • 特殊字符: 具有功能的特殊字符，java字符串中使用时需要进行转义。
    • 限定字符: 限定表达式出现的次数。
  • RE正则匹配:
    • str.matches("RE"): 对字符串str进行RE匹配，返回布尔型结果。
  • RE正则替换:
    • str.replaceAll("RE", "*"): 将字符串str中满足RE规则的内容全部替换为 "*" 并返回替换后的字符串。
  • RE正则切割:
    • str.split("RE"): 将字符串str按照RE规则拆分成字符串数组，结果中不含RE内容。

CH10. 数据结构之数组

心法: 数据结构之数组

  • 数组是heap内存中一段内存连续的线性数据结构，属于引用数据类型:
    • 数组的下标从0开始，支持使用 数组[索引] 的格式来操作数组元素。
    • 数组中的每个最底层元素都具有默认值，如 0, 0.0, false, \u0000, null 等。
    • 数组在创建的时候，要么指定内容，要么指定长度，但二者不可同时指定。
  • 数组增删慢:
    • 数组长度不可改变，导致增删操作必须使用创新数组，效率低。
  • 数据查询快:
    • 数组中每个元素的内存地址连续可计算，所以在明确位置的情况下，查询效率更高。

EP01. 单维度数组结构

    • 对一维字符数组拼接空串或调用其 toString() 可获取其内存地址。

EP03. 多维度数组结构

心法: 多维度数组结构，以二维数组为例

  • 二维数组的本质也是一维数组，只不过其中的每个元素都是一个一维数组类型：
    • 二维数组中元素的类型必须一致，但元素的长度可以不同。
    • 二维数组本身是内存连续的，但其指向的N个一维数组未必是内存连续的。
  • 直接输出二维数组，打印的是该数组的内存地址，即使是字符数组也一样。
