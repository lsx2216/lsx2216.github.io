S01. Vue基础入门

E01. Vue概念入门

心法: Vue

  • Vue是一套用于构建用户界面的渐进式框架，Vue组件使用 .vue 作为文件后缀:
    • Vue文件使用 <template> 划分HTML代码开发区。
    • Vue文件使用 <style> 划分CSS/SCSS代码开发区。
    • Vue文件使用 <script> 划分JS代码开发区。
  • Vue底层采用MVVM架构，不允许直接操作DOM树，而是采用数据驱动DOM的方式工作。
MVVM架构对比MVC架构描述代码所属区域
Model模型层负责定义数据<script>
ViewModel控制层负责连接V层和M层<script>
View视图层负责DOM元素的开发<template> + <style>

E05. Vue响应变量

心法: ref() 函数

  • ref() 函数来声明响应式JS变量，即JS变量发生变化时，页面端会立刻同步最新值。
  • ref() 的参数可以是任何类型，如string，number，boolean，JSON对象，数组，Map或Set等。
  • ref() 函数会将其参数封装到一个 {value: 参数} 中:
    • 获取和设置ref变量时，必须使用 变量.value 的格式。
    • HTML中会自动解包，无需使用 变量.value 的格式，直接使用变量名即可。
  • shallowRef(): 浅层次的 ref()，即仅对第一层数据添加响应式效果，节省开销。

武技: 测试 ref() 函数

  • template
<h1>无响应式: {{ numA }}</h1>
<button type="button" @click="incrementA">无响应式自增</button>

<h1>有响应式: {{ numB }}</h1>
<button type="button" @click="incrementB">有响应式自增</button>
  • script
import {ref} from "vue";

let numA = 0;
let numB = ref(0);

function incrementA() {
  numA++;
  console.log(numA);
}

function incrementB() {
  numB.value++;
  console.log(numB.value);
}

E06. Vue响应对象

  • shallowReactive(): 浅层次的 shallow()，即仅对第一层数据添加响应式效果，节省开销。

E07. Vue计算属性

心法: computed() 函数 - 计算属性

  • computed() 函数用于对返回一个经过复杂处理后的值。
  • computed() 函数的本质是属性，而不是方法，所有在HTML中调用的时候不要携带小括号。
  • computed() 在调用时可以传递参数，但设计计算属性时需要闭包，然后在内层函数中接受参数。
  • computed() 对比 function() 多了缓存的功能，只要值不变，则无需每次都重新计算一遍，节省计算开销。
  • 在JS中调用计算属性的值，需要使用 计算属性.value 来获取。

武技: 测试 computed() 函数

  • template
<h1>{{ genderComputed }}</h1>
<h1>{{ priceComputed(9.9) }}</h1>
<button @click="print">在JS中输出计算属性</button>
  • script
import {computed} from "vue";

let gender = 0;

// 不带参数调用计算属性
let genderComputed = computed(() => {
  if (gender === 0) {
    return '女';
  } else if (gender === 1) {
    return '男';
  } else {
    return '保密';
  }
});

// 带参数调用计算属性，需要闭包
let priceComputed = computed(() => (price) => price + '元');

// 输出计算属性的值
function print(){
  console.log(genderComputed.value);
  console.log(priceComputed.value(100));
}

E12. Vue生命周期

心法: Vue生命周期函数

  • 每个Vue组件实例在创建时都需要经历一系列的初始化步骤，比如设置好数据侦听，编译模板，挂载实例到DOM，以及在数据改变时更新DOM。
  • 在此过程中，它也会运行被称为生命周期钩子的函数，让开发者有机会在特定阶段运行自己的代码。

生命周期函数描述
onBeforeMount()在组件被挂载之前调用
onMounted()在组件挂载完成后执行，当且仅当全部子组件和本身都挂载完毕时视为挂载完成
onBeforeUpdate()在组件即将因为响应式状态变更而更新其 DOM 树之前调用
onUpdated()在组件因为响应式状态变更而更新其 DOM 树之后调用父组件的 onUpdated() 将在其子组件的 onUpdated() 之后调用
onBeforeUnmount()在组件实例被卸载之前调用
onUnmounted()在组件实例被卸载之后调用，当且仅当全部子组件和本身都卸载完毕时视为卸载完成卸载前需要停止组件的全部响应式作用，如计算属性，侦听器等

武技: 测试Vue生命周期函数

  • template
<p>{{ message }}</p>
  • script
import {onBeforeMount, onMounted, ref} from "vue";

let message = ref('');

/*当组件加载前执行*/
onBeforeMount(() => {
  console.log('onBeforeMount -> message: ' + message.value);
  message.value = "Hello"
});

/*当组件加载后执行*/
onMounted(() => {
  console.log('onMounted -> message: ' + message.value);
  message.value = "World";

E13. Vue属性监听

心法: Watch监听器

  • watch(JS变量, (原值, 新值) => {})): 用来监听某个JS变量，一旦该JS变量发生改变，则触发该监听器函数。
    • 允许直接监听一个 Ref 变量。
    • 允许直接监听一个 Computed 属性。
    • 不允许直接监听一个 Reactive 变量，需要将其处理为一个函数的返回值 () => 变量，然后再进行监听。

武技: 测试Watch监听器

let num = ref(0);

E14. Vue父子组件

心法: Vue组件

  • Vue组件允许我们将 UI 划分为独立的、可重用的部分，并且可以对每个部分进行单独的思考。
  • 父组件引入子组件（该子组件须配置正确的路由）:
    • 父组件通过 import 组件名 from '组件地址' 的方式在JS区中引入子组件。
    • 父组件通过 <script setup> 导入的组件在模板中可以直接使用，无需注册。
    • 父组件可以直接在 <template> 中，通过子组件的名称，或其连字符格式的名称来使用子组件。
  • 父组件可以向子组件传递数据:
    • 父组件使用 <子组件 自定义属性名 = "值"> 的格式向子组件传递数据。
    • 子组件使用 defineProps(['自定义属性名']) 的格式接收父组件传递过来的数据。

E04. 安装Axios

心法: Axios工具组件用于向服务器发送请求，是AJAX的一种实现

  • Axios 是一个基于 Promise 的 HTTP 客户端，用于浏览器和 Node.js 环境中发起 HTTP 请求。它是一个流行的工具，用于与后端 API 进行通信，提供了许多便捷的功能和选项，如异步请求、拦截器、取消请求等。
优点描述
Promise BasedAxios 基于 Promise，使得在处理异步请求时更加简洁和灵活
兼容浏览器和Node.jsAxios 可以在浏览器和 Node.js 环境中使用，因此适用于前端和后端开发
易于使用Axios 支持各种 HTTP 请求方法，如 GET、POST、PUT、DELETE 等
拦截器Axios 允许在请求或响应被发送或接收之前进行拦截和转换，这为添加全局请求和响应逻辑提供了便利
取消请求Axios 允许通过取消令牌来取消请求，防止不必要的请求发送和资源浪费
防止 CSRFAxios 支持在客户端中防止 CSRF（跨站请求伪造）攻击，通过设置 withCredentials 选项来发送跨站点凭证
自动转换 JSONAxios 会自动将请求和响应的数据转换为 JSON 格式，使得与后端 API 交互更加便捷
错误处理Axios 提供了简单易用的错误处理机制，可以捕获和处理请求和响应的错误
并发请求Axios 允许同时发起多个并发请求，并且提供了处理并发请求的便捷方式

武技: 安装Axios

1. 发送GET请求

let url = 'http://localhost:8080/api/v1/test';
let param = {name: 'lsx', age: 18};

// 后台无参数
axios.get(url);

// 后台有参数：参数标记 @RequestParam 注解或无注解
axios.get(url,  { params: { param } });

// 后台有参数：参数标记 @PathVariable 注解
axios.get(url + '/lsx/18');

2. 发送POST请求

let url = 'http://localhost:8080/api/v1/test';
let param = {name: 'lsx', age: 18};

// 后台无参数
axios.post(url);

// 后台有参数：参数标记 @RequestParam 注解或无注解
// qs工具需要导入
axios.post(url, qs.stringify(param), { headers: {'Content-Type':'application/x-www-form-urlencoded'} });

// 后台有参数：参数标记 @RequestBody 注解
// headers['Content-Type'] 的默认值就是 'application/json':
axios.post(url, param, { headers: {'Content-Type':'application/json'} });

// 后台有参数：参数为 MutipartFile 类型
let formData = new FormData();
formData.append('id', 9527);
formData.append('filename', 'test.jpg');
axios.post(url, formData, { headers: {'Content-Type':'multipart/form-data'} });
